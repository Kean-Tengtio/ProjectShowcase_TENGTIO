How Does it Work?

Proxy Core

Corefuse creates a virtual core that will act as the main core for the application.
This core will pool power equally from every core on the actual CPU for it to function.
Of note is that it simply reserves a portion from each one, rather than constantly taking power.

Activation & Deactivation

Corefuse only activates when allowed, and, by default, will not activate on any application.
The user will be told this beforehand. The application will bring up a list on first start-up
of applications it considers valid, though the user can manually add them on their own.
Once the program it is running for closes, it will either return to idle or close entirely,
depending on its settings.
